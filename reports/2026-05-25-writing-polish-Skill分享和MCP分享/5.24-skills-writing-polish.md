# 5.24 Skills: Academic Writing Polish

## 简介

`academic-anti-ai-polisher` 是一个面向学术写作的 Codex skill，用于降低论文、综述、研究报告、开题材料和学术段落中的模板化 AI 写作痕迹。

它的目标不是把学术文本改得更口语、更夸张或更像随笔，而是在保留原始含义和学术严谨性的基础上，让文字更具体、更自然、更克制，也更像真实研究者写出的学术表达。

这个 skill 特别关注一个问题：很多 AI 生成文本看起来流畅，但经常空泛、拔高、重复、过度使用连接词，并且喜欢用“重要意义”“关键作用”“广阔前景”这类没有信息量的表达。`academic-anti-ai-polisher` 的作用就是把这些表层顺滑但内容稀薄的句子，改成更有证据感和分析密度的学术表述。

## 核心目标

这个 skill 的核心目标可以概括为：

> 在不改变原文技术含义、引用关系、术语和论证边界的前提下，减少学术文本中的模板化、宣传化和空泛化表达。

它重点保护以下内容：

1. 原始技术含义
2. 术语一致性
3. 引用和证据对应关系
4. claim 的强弱边界
5. 段落的论证结构
6. 正式、克制的学术语气

## 适合处理的文本

这个 skill 适合处理：

- 论文摘要
- 引言段落
- 文献综述
- 相关工作
- 方法描述
- 实验分析
- 讨论部分
- 未来工作
- 学术报告和研究计划

典型使用请求可以写成：

```text
使用 $academic-anti-ai-polisher 润色下面这段学术文本，降低 AI 味，但不要改变技术含义和引用关系。
```

```text
帮我把这段综述改得更自然、更具体，删掉空泛和模板化表达。
```

```text
请检查下面这段文字是否有 AI 写作痕迹，并给出学术化改写版本。
```

## 它会处理哪些问题

### 1. 空泛拔高

AI 生成的学术文本经常喜欢把普通发现拔高成宏大叙述。典型表现是**反复使用“重要意义”“关键作用”“深远影响”“广泛关注”等说法，但没有说明具体证据**。

**处理原则：不要只说“重要”，而要说明它为什么重要、在哪里重要、通过什么证据体现出来。**

例子 1：

```text
改写前：
人机协作在现代社会中具有重要意义，并在人类工作方式转型中发挥着关键作用。

改写后：
在人机协作场景中，AI 系统通常承担信息检索、初稿生成和重复性判断等任务，而人类负责目标设定、结果审核和责任判断。
```

例子 2：

```text
改写前：
智能助手的出现标志着人机关系进入了一个全新的发展阶段。

改写后：
智能助手使用户可以用自然语言完成检索、写作和日程管理等任务，但用户仍需要检查系统输出是否准确。
```

例子 3：

```text
改写前：
人机交互研究受到了学术界和产业界的广泛关注，具有深远的理论和实践价值。

改写后：
人机交互研究关注用户如何理解系统反馈、如何纠正系统错误，以及如何在高风险任务中分配人与机器的决策权。
```

### 2. 宣传式或过度包装语言

学术写作需要中性、可验证的表达，不需要广告式包装。**AI 文本常把普通功能写成“突破性”“革命性”“强大”“无缝”等。**

**处理原则：把夸张评价改成具体功能、适用范围、实验结果或限制条件。**

例子 1：

```text
改写前：
该智能助手提供了强大而无缝的人机协作体验。

改写后：
该智能助手可以根据用户指令生成任务清单，并在用户确认后将任务同步到日历。
```

例子 2：

```text
改写前：
这项技术将彻底改变人类与机器互动的方式。

改写后：
这项技术允许用户通过语音指令控制设备，但在嘈杂环境中的识别准确率仍然下降明显。
```

例子 3：

```text
改写前：
该系统展现出革命性的潜力，为人机关系带来了前所未有的机遇。

改写后：
该系统降低了用户完成表单填写任务所需的点击次数，但没有评估用户对自动填充结果的信任程度。
```

### 3. 固定套路结构

**AI 文本经常使用整齐但僵硬的句式，让段落显得像模板。**常见结构包括“不仅……而且……”“不只是……更是……”“从……到……，从……到……”。

**处理原则：如果两个意思都重要，就直接说明它们之间的关系；如果只是为了制造气势，就删掉套路结构。**

例子 1：

```text
改写前：
AI 不仅是人类工作的辅助工具，更是重塑人机关系的重要力量。

改写后：
AI 可以辅助人类完成资料整理、文本生成和初步判断，但任务目标和最终责任仍由人类决定。
```

例子 2：

```text
改写前：
人机协作不只是效率提升的问题，更是社会结构变化的重要体现。

改写后：
人机协作会改变部分岗位的任务分配，例如将初稿撰写交给系统，将审核和修改留给人工。
```

例子 3：

```text
改写前：
从家庭生活到工作场景，从教育培训到医疗服务，智能系统正在全面融入人类社会。

改写后：
智能系统已经被用于家庭语音控制、办公写作辅助、在线学习反馈和医疗影像初筛等场景。
```

### 4. 模糊归因

“研究表明”“专家认为”“已有工作指出”本身不是问题，但如果**没有明确证据**，就会显得空泛，也容易把有限证据写成普遍结论。

**处理原则：明确谁的研究、什么任务、什么证据、支持到什么程度。不能把少量观察写成确定共识。**

例子 1：

```text
改写前：
大量研究表明，人类越来越依赖人工智能系统。

改写后：
部分问卷研究发现，受访者在搜索、翻译和写作任务中使用 AI 工具的频率较高，但这些研究通常依赖自我报告数据。
```

例子 2：

```text
改写前：
专家认为，智能助手能够显著改善人类决策质量。

改写后：
在若干受控实验中，智能助手可以帮助用户更快发现备选方案，但当系统解释不充分时，用户可能过度采纳错误建议。
```

例子 3：

```text
改写前：
已有研究证明，人机协作比单独依靠人类或机器更有效。

改写后：
已有研究比较了人工决策、机器决策和人机协作三种模式；结果显示，人机协作的效果取决于任务难度、系统准确率和用户是否理解系统建议。
```

### 5. 连接词和填充语堆叠

AI 文本**喜欢用连接词制造“有条理”的感觉**，但过多连接词会让段落显得机械。常见表达包括“此外”“同时”“值得注意的是”“在这一背景下”“总体而言”。

**处理原则：连接词只在真正表达逻辑关系时保留。没有必要的连接词可以直接删除。**

例子 1：

```text
改写前：
此外，值得注意的是，智能客服在提升用户体验方面发挥了重要作用。

改写后：
智能客服可以回答常见问题并分流简单请求，但复杂投诉仍需要人工客服处理。
```

例子 2：

```text
改写前：
在这一背景下，人机关系的变化成为一个值得深入探讨的重要问题。

改写后：
随着生成式 AI 被用于写作和编程，用户需要判断哪些任务可以交给系统，哪些任务仍应由自己完成。
```

例子 3：

```text
改写前：
总体而言，AI 的发展为人类社会带来了新的机遇和挑战。

改写后：
AI 工具可以减少重复性劳动，但也会带来技能退化、责任归属和错误依赖等问题。
```

### 6. 空洞分析动词

**AI 文本常用一些看似学术但不够具体的动词**，如“强调”“凸显”“彰显”“展示”“促进”“赋能”“推动”。这些词如果没有具体对象和机制，就会显得空。

**处理原则：把这些动词换成真实动作或关系，例如“区分”“比较”“估计”“验证”“降低”“提高”“依赖于”。**

例子 1：

```text
改写前：
该研究凸显了信任在人机关系中的重要性。

改写后：
该研究将用户信任分为能力信任、过程信任和责任信任，并分别测量它们对系统采纳意愿的影响。
```

例子 2：

```text
改写前：
智能系统赋能用户完成更加复杂的任务。

改写后：
智能系统通过提供候选答案和错误提示，帮助用户完成原本需要多轮检索的信息整理任务。
```

例子 3：

```text
改写前：
人机协作机制推动了组织效率的提升。

改写后：
在人机协作流程中，系统先对用户请求进行分类，再将低风险请求自动处理，将高风险请求转交人工审核。
```

### 7. 空泛结尾

AI 文本很容易用**漂亮但没有信息量**的句子收尾，例如“未来前景广阔”“仍有很大探索空间”“为相关研究提供重要启示”。

**处理原则：结尾应该指出具体限制、具体问题或具体下一步，而不是泛泛说“未来很好”。**

例子 1：

```text
改写前：
未来，人机关系研究仍有广阔的发展空间。

改写后：
未来研究需要进一步比较不同用户群体在接受 AI 建议时的信任差异，例如学生、医生和客服人员是否采用相同的判断标准。
```

例子 2：

```text
改写前：
该研究为理解智能时代的人机关系提供了重要启示。

改写后：
该研究表明，用户是否采纳 AI 建议不仅取决于系统准确率，还取决于系统是否解释其判断依据。
```

例子 3：

```text
改写前：
随着技术不断进步，人机协作将展现出更加广阔的应用前景。

改写后：
后续工作可以评估人机协作在高风险任务中的适用边界，尤其是医疗诊断、金融审核和公共服务决策等场景。
```

## 不可违反的约束

使用这个 skill 时，需要遵守以下约束：

- 不改变原文的技术含义。
- 不改变原文的引用关系。
- 不新增没有证据支持的结论。
- 不随意增加数字、实验结果、机制解释或比较对象。
- 不删除必要的专业术语。
- 不把不确定结论改成确定结论。
- 不把正式学术文本改成营销文、新闻稿或随笔。
- 不为了“自然”而牺牲准确性。

这些约束很重要。降低 AI 味不是把文本写得更花，而是把空话删掉，把具体信息讲清楚。

## 总结

`academic-anti-ai-polisher` 的重点不是“让文字更华丽”，而是“让文字更具体、更稳、更有证据边界”。它通过删除空泛拔高、宣传式措辞、固定套路句式和无信息量结尾，让学术文本从“看似流畅”变成“真正清楚”。
