# 笑话大王

## 目标

根据用户需求选择笑话类型并生成对应的交付文件：

- 冷笑话：生成精美 `html`或 `pdf`
- 段子：生成精美 `html`或 `pdf`

## 渐进式流程（必须遵循）

1. 判断用户想要的笑话类型：`冷笑话` / `段子`
2. 若用户未明确类型：向用户追问并确认
3. 确认导出格式：`HTML` / `PDF`
4. 按类型分支执行

## 追问模板（用户没说清时）

请问你想要哪种笑话？

- 冷笑话（一本正经的尬）
- 段子（小故事/对话型）

另外有两个可选信息：

- 主题（比如：程序员、上班、学习、猫）
- 条数（默认 3 条）
- 标题（可选，用于导出的 HTML/PDF 标题）
- 输出路径（可选，默认在会话目录生成）

## 冷笑话分支

### 参考资料（必须先读）

读取 `Reference/冷笑话参考.md`，参考其风格写出新的冷笑话，不要参考其文本内容，内容需要你自己原创。每条冷笑话必须以以下前缀开头：

`冷笑话：   `

### 生成脚本

在生成交付物时，你需要先自己原创生成冷笑话文本（参考风格但不要复用参考文本），把生成的冷笑话写入一个临时文本文件，然后执行：

```bash
python Scripts/render_html.py --kind "cold" --in "<冷笑话文本文件绝对路径>" --out "<输出路径>.html" --title "<标题可选>"
```

要求：

- 输出文件必须是 UTF-8 编码
- 每条以 `冷笑话：   ` 开头
- 每条冷笑话以 `冷笑话：` 开头；同一条内容允许换行续写，但不同条之间用空行分隔
- 最终输出物为冷笑话文本内容和一个 html 文件

如果用户要求导出 PDF，则执行：

```bash
python Scripts/render_pdf.py --kind "cold" --in "<冷笑话文本文件绝对路径>" --out "<输出路径>.pdf" --title "<标题可选>"
```

## 段子分支

读取 `Reference/段子参考.md`，参考其结构与节奏写出新的段子，不要参考其文本内容，内容需要你自己原创。每个段子必须以以下前缀开头：

`段子：  `


- 每条段子以 `段子：` 开头；同一条内容允许换行续写，但不同条之间用空行分隔
- 最终输出物为段子文本内容和一个 html 文件

如果用户要求导出 PDF，则执行：

```bash
python Scripts/render_pdf.py --kind "duanzi" --in "<段子文本文件绝对路径>" --out "<输出路径>.pdf" --title "<标题可选>"
```
